Claudeの特徴 — 4つのポイント

1. 自律的に稼働する

指示に対して自ら考え、必要な手順を組み立てて実行。逐一の指示は不要。

2. あらゆるアウトプットを作成

文章・スライド・Excel・画像・アプリケーションなど、成果物を直接作成。

3. 外部アプリと連携

データのやりとり・情報の探索・活用が可能。連携方法も自律的に探索。

4. 記憶を引き継ぐ

過去のデータや作業を記憶し、文脈を保持したまま継続的な作業が可能。

「毎回イチから説明しなくていい」のが実務では大きい
例: 「先週の議事録と同じフォーマットで今週分も作って」が通じる

安全性・セキュリティ

入力データはモデルの学習に使用されない（Teamプラン）

アクセス範囲はユーザーが限定・制御できる

国際セキュリティ認証を取得（SOC 2 / ISO 27001 / ISO/IEC 42001）

開発元 Anthropic社は元OpenAI研究者が「AI安全性を最優先」に設立。
Google・Amazon・Salesforceが出資。

AIの回答を鵜呑みにしないこと。最終確認は必ず人間が行う

フォルダ構成

├── A社/（クライアントA）
│ ├── 案件A/

├── B社/（クライアントB）
├── Other/（個人案件）
├── CLAUDE.md（共通ルール）
└── TASKS.md（横断タスク管理）

AIに任せる仕事 vs 人間がやる仕事

AIに任せてOK

  • 下書き作成（メール・報告書・議事録）
  • 情報の整理・要約・翻訳
  • データの集計・グラフ化
  • 定型フォーマットへの変換
  • アイデア出し・壁打ち

人間が必ずやること

  • 最終判断・意思決定
  • 対外コミュニケーション（送信前確認）
  • 数字・事実の正確性チェック
  • 機密情報の取扱い判断
  • 成果物の最終承認

AIは「優秀だけど入社1年目のアシスタント」
下書きは頼む。でも提出前に必ず自分の目で確認する。

注意してほしいこと

⚠ AIは間違えることがある（ハルシネーション）

重要な数字・事実は必ず自分で確認する。「もっともらしい嘘」が一番危険。

⚠ 個人情報・極秘情報の取扱い

Teamプランではデータ学習に使われないが、組織のルールに従って運用する。

⚠ AIの回答を「自分の意見」として出す前に確認

コピペして提出する前に必ず内容を読む。

Claude Code — 何が変わるのか

Coworkでの作業:

あなた:「議事録を作って」
Claude:「はい、以下が議事録です」
あなた:（コピー → Word貼付 → 保存…）

Claude Codeでの作業:

あなた:「議事録を作ってWord形式で保存して」
Claude Code: メモ読取 → 議事録作成 → 保存 → 完了報告
あなた:（フォルダにファイルがある！）

コピペ・保存・整理の手間がゼロ。「PCにもう一人アシスタントがいる」感覚

Mac

方法B: Finder →「アプリケーション」→「ユーティリティ」→「ターミナル」

Windows

方法A: スタートメニュー →「PowerShell」→ Enter

方法B: Windowsキー+X → 「PowerShell」を選択

怖くない！ ただの「文字で操作する画面」です

ハンズオン② ファイル作成

Claude Codeに以下を入力:

このフォルダに「自己紹介.txt」というファイルを作ってください。

内容:

名前: （自分の名前）

部署: （自分の部署）

今日学んだこと: Claude Codeの使い方

「作成してもよいですか？」
→ 内容を確認して「許可」を押す

確認コマンド:
Mac: ls ~/claude-work/
Win: dir ~\claude-work\

安全の仕組み — 全操作に「許可」が必要

ファイル作成 → 「作っていいですか？」 → あなたが許可

ファイル編集 → 「この部分を変えていいですか？」 → あなたが許可

コマンド実行 → 「実行していいですか？」 → あなたが許可

許可の判断基準:

  • 何をするか表示される → 読んで理解できる
  • 問題あれば「いいえ」で中止できる
  • 操作範囲は指定したフォルダ内に限定

「優秀だけど、何をする前にも必ず報告してくる新入社員」のイメージ
